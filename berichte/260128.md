## Übung 3.3 (Wildcards)

### * : Platzhalter für beliebig viele Zeichen
### ? : Platzhalter für ein Character 
### [] : Man kann in die Klammer die Auswahl für den Platzehalter schreiben
### {} : Man kann in die Klammer die Endungen schreiben

## Übung 3.4 (find)

#### https://www.geeksforgeeks.org/linux-unix/find-command-in-linux-with-examples/

```sh
find -name "profiles"
find -name ".latest.json"
find -name "*.json"
find type d
find -name "*.md" -mtime -28
find -type d -mtime -28
find -type d -atime -28
find -type f -perm 644
```

### Linux-Permissions (Berechtigungen) definieren, wer Dateien und Verzeichnisse lesen (r), schreiben (w) oder ausführen (x) darf, unterteilt für den Besitzer (user/u), die Gruppe (group/g) und alle anderen (others/o), um die Systemsicherheit zu gewährleisten.
### 644 bedeutet, dass nur der Besitzer der Datei die Datei ändern kann und andere nur lesen könen

```sh
find /usr -type f -size +2M
```

```sh
find -size -10c -exec rm {} \;
``` 

```sh
find -name "*.md" -exec cat {} +  >newfile.md
```  

```sh
find -name "*log*.*" -ls
```

## Übung 3.6 (cat)

```sh
cat -n Dateiname

ODER

cat -b Dateiname
```

```
Die "-n" Option numeriert auch leere Zeilen (exkl. die letzte Zeile falls sie leer ist)
Die "-b" Option überspringt sozusagen die leeren Zeilen
Die Option "-E" ist dabei ein nützliches Werkzeug zur Fehlersuche (Debugging), da sie unsichtbare Zeichen sichtbar macht. 
```
