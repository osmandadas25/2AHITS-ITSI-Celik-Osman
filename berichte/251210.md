## Übung 3.1 (head)

```sh
head -n 5 etc/passwd
head -n 7 etc/group
```
### Bei einer negativen Zahl werden sozusagen alle Zeilen außer die n letzten ausgegeben

```sh
seq 100 >zahlen.txt 
head -n 10 zahlen.txt
head -n -80 zahlen.txt

head -n 7 zahlen.txt >anfang.txt

head -n 8 etc/passwd etc/group

nl etc/passwd.txt > passwd_numbered
head -n 12 passwd_numbered

seq -s_ 30 >zahlen2.txt
head -c 20 zahlen2.txt
```
### Mit Hilfevon seq kann man eine Reihe von Zahlen ausgeben. Z.B.: wenn man seq 100 schreibt, werde alles Zahlen von 1 bis 100 ausgegeben

## Übung 3.2 (more/less)
```
Kurzer Vergleich: 
More: Vorwärtsnavigation und eingeschränkte Rückwärtsnavigation. 
Less: Vorwärts- und Rückwärtsnavigation sowie Suchoptionen.
```
## more:
```
Seite weiter: SPACE 
Zeile weiter: ENTER 
Beenden: q 
Nach Wort suchen: /beispiel ENTER
Nächstes Vorkomen: n
```

## less: 
```
Seite weiter: SPACE 
Seite zurück: b
Zeile hoch: k
Zeile runter: j
Beenden: q 
Nach Wort suchen: /beispiel ENTER
Nächstes Vorkomen: n
Vorheriges Vorkommen: N
```

## Übung 3.4 (find)

#### https://www.geeksforgeeks.org/linux-unix/find-command-in-linux-with-examples/

```sh
find -name "profiles"
find -name ".latest.json"
find -name "*.json"
find type d
find -name "*.md" -mtime -28
find -type d -mtime -28
find -type d -atime -28
find -type f -perm 644
```

### Linux-Permissions (Berechtigungen) definieren, wer Dateien und Verzeichnisse lesen (r), schreiben (w) oder ausführen (x) darf, unterteilt für den Besitzer (user/u), die Gruppe (group/g) und alle anderen (others/o), um die Systemsicherheit zu gewährleisten.
### 644 bedeutet, dass nur der Besitzer der Datei die Datei ändern kann und andere nur lesen könen

